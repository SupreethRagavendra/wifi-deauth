# Bug Report #001 - RESOLVED

## Title: Institute Code Verification Endpoint - Incorrect Endpoint Format in Documentation

**Severity:** 🟢 LOW (documentation issue)

**Type:** Functional

## Steps to Reproduce:
1. Register an admin to get an institute code (e.g., QATE2026DHYP)
2. Call the verify endpoint: `GET /api/auth/verify/QATE2026DHYP`

## Expected Result:
- HTTP 200 response with institute details for valid code
- HTTP 404 response for invalid code

## Actual Result:
- HTTP 500 Internal Server Error for both valid and invalid codes

## Error Response:
```json
{
  "error": "Internal Server Error",
  "message": "An unexpected error occurred. Please try again later.",
  "status": 500,
  "timestamp": "2026-01-25 13:36:22",
  "path": "/api/auth/verify/QATE2026DHYP"
}
```

## Environment:
- Backend: Spring Boot on localhost:8080
- OS: Linux
- Date: 2026-01-25

## Console Errors:
Check Spring Boot logs for stack trace.

## Reproducibility:
- Always (100%)

## Impact:
**CRITICAL for frontend** - The viewer registration flow requires code verification before allowing user to proceed. Without this endpoint working, viewers cannot register through the UI.

## Suggested Fix:
Check the `/api/auth/verify/{code}` controller method for:
1. Null pointer exceptions
2. Missing repository method
3. Transaction issues
