# Bug Report #002

## Title: Frontend Uses Wrong Endpoint for Institute Code Verification

**Severity:** 🔴 CRITICAL (completely blocks viewer registration)

**Type:** Functional / Integration

## Steps to Reproduce:
1. Navigate to http://localhost:3000/register
2. Click on "Organization Viewer" option
3. Enter a valid institute code (e.g., FUNC20265K3W)
4. Click "Verify Code" button

## Expected Result:
- Institute code should be verified successfully
- Institute name should be displayed
- User should be able to proceed with registration

## Actual Result:
- Error message: "Verification Failed - Invalid institute code"
- User cannot proceed with registration
- Console shows 403 Forbidden error for GET request to `/institutes/verify/{code}`

## Root Cause Analysis:
**Frontend code** (`/wifi-security-frontend/src/services/api.ts` line 176):
```typescript
const response = await api.get(`/institutes/verify/${code}`);
```

**Backend expects** (`/wifi-security-backend/src/main/java/com/wifi/security/controller/AuthController.java`):
```java
@PostMapping("/verify-institute-code")
public ResponseEntity<InstituteVerificationResponse> verifyInstituteCode(
    @RequestBody Map<String, String> request
)
```

## Mismatch:
| Aspect | Frontend (Current) | Backend (Expected) |
|--------|-------------------|-------------------|
| Method | GET | POST |
| Endpoint | /institutes/verify/{code} | /auth/verify-institute-code |
| Code Location | URL path parameter | Request body JSON |

## Impact:
- **ALL Organization Viewer registrations are blocked**
- Users cannot join existing institutes
- Feature is completely broken in UI (works via API only)

## Verification:
Backend API works correctly when called properly:
```bash
curl -X POST http://localhost:8080/api/auth/verify-institute-code \
  -H "Content-Type: application/json" \
  -d '{"instituteCode": "FUNC20265K3W"}'

# Response: {"valid":true,"instituteName":"Functional Test College","instituteType":"COLLEGE"}
```

## Fix Required:
Update `/wifi-security-frontend/src/services/api.ts` line 172-186:
- Change from `GET /institutes/verify/${code}` to `POST /auth/verify-institute-code`
- Send code in request body instead of URL
